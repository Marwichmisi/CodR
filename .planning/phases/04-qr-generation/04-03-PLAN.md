<task type="auto">
  <name>Task 2: Refactor generator_screen.dart to use PermissionService</name>
  <files>qr_scanner/lib/screens/generator_screen.dart</files>
  <action>
Replace the direct `Permission.photos.request()` call in `_saveToGallery()` (line 207) with `PermissionService.requestGalleryPermission()`.

Steps:
1. Add import for PermissionService: `import '../services/permission_service.dart';`
2. Add a `PermissionService` field to `_GeneratorScreenState` and inject it via constructor parameter or accept it from `GeneratorScreen`. Since `GeneratorScreen` already takes a `GeneratorViewModel`, the simplest approach is to pass `PermissionService` as a second required parameter to `GeneratorScreen` and store it in state.
3. In `_saveToGallery()`, replace `final status = await Permission.photos.request();` with `final granted = await permissionService.requestGalleryPermission();` and adjust the conditional accordingly (use `!granted` instead of `!status.isGranted`).
4. Remove the direct `import 'package:permission_handler/permission_handler.dart';` since PermissionService handles it internally.
5. Update `app_router.dart` to pass the existing `PermissionService` instance when constructing `GeneratorScreen`.
6. Update widget tests and preview mocks to include the new `permissionService` parameter.

The `PermissionService.requestGalleryPermission()` already handles the iOS/Android permission split (Permission.photos first, then Permission.storage fallback) per the existing implementation in permission_service.dart.
  </action>

  </verify>
  <done>generator_screen.dart uses PermissionService.requestGalleryPermission() instead of direct Permission.photos.request(). No direct permission_handler imports in generator_screen.dart. All tests pass.</done>
