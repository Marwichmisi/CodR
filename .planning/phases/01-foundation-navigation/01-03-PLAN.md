The `StorageService` class must:

1. **Singleton pattern**: Static `_database` field, async `database` getter that lazily initializes
2. **Database initialization** (`_initDatabase`):
   - Use `getDatabasesPath()` from sqflite
   - Join path with `'qr_scanner.db'` using `path` package
   - Call `openDatabase` with `version: 1` and `onCreate: _createDatabase`
3. **Table creation** (`_createDatabase`):
   - Create `scan_records` table: `id INTEGER PRIMARY KEY AUTOINCREMENT, content TEXT NOT NULL, timestamp TEXT NOT NULL, type TEXT NOT NULL`
   - Create `generation_records` table: same schema
   - Both tables in the SAME database file (SPEC prohibition R5)
4. **CRUD methods**:
   - `Future<void> insertRecord(String table, Map<String, dynamic> data)` — uses `db.insert(table, data)`
   - `Future<List<Map<String, dynamic>>> getAll(String table)` — uses `db.query(table, orderBy: 'timestamp DESC')`
   - `Future<Map<String, dynamic>?> getById(String table, int id)` — uses `db.query` with `where: 'id = ?', whereArgs: [id]`
   - `Future<void> delete(String table, int id)` — uses `db.delete` with `where: 'id = ?', whereArgs: [id]`
5. **Helper methods** for model-specific operations:
   - `Future<void> insertScanRecord(ScanRecord record)` — converts to JSON, inserts into 'scan_records'
   - `Future<void> insertGenerationRecord(GenerationRecord record)` — converts to JSON, inserts into 'generation_records'
   - `Future<List<ScanRecord>> getAllScanRecords()` — queries 'scan_records', maps to ScanRecord via fromJson
   - `Future<List<GenerationRecord>> getAllGenerationRecords()` — queries 'generation_records', maps to GenerationRecord via fromJson

Key implementation details per RESEARCH.md:
- All queries use parameterized `whereArgs` (not string interpolation) to prevent injection per security domain
- DateTime stored as ISO 8601 TEXT via `toIso8601String()` per Pitfall 5
- Import `package:sqflite/sqflite.dart` and `package:path/path.dart`
- Import the two model classes from `lib/models/`

Add `WidgetsFlutterBinding.ensureInitialized()` in `main()` per Pitfall 4 (already needed for sqflite).

```dart
LayoutBuilder(
  builder: (context, constraints) {
    final maxWidth = constraints.maxWidth > 600 ? 480.0 : constraints.maxWidth;
    return Scaffold(
      appBar: AppBar(title: const Text('QR Scanner')),
      body: Center(
        child: ConstrainedBox(
          constraints: BoxConstraints(maxWidth: maxWidth),
          child: Column(
            mainAxisAlignment: MainAxisAlignment.center,
            children: [
              Icon(icon, size: 64, color: Theme.of(context).colorScheme.primary),
              const SizedBox(height: 32),
              Text(title, style: Theme.of(context).textTheme.displayMedium),
              const SizedBox(height: 16),
              Text(subtitle, style: Theme.of(context).textTheme.bodyLarge),
            ],
          ),
        ),
      ),
    );
  },
)
```

Key responsive rules per UI-SPEC contract:
- Phone (< 600px): `maxWidth = constraints.maxWidth` (full width)
- Tablet (>= 600px): `maxWidth = 480.0` (constrained and centered)
- No orientation locking per `flutter-build-responsive-layout` skill
- No hardcoded pixel values — use `constraints.maxWidth` from `LayoutBuilder`

Per `flutter-fix-layout-issues` skill: The `Column` inside `Center` is safe because `Center` provides bounded constraints. The `ConstrainedBox` prevents content from stretching on wide screens. No `Expanded` needed since the content is a fixed-height `Column`.
