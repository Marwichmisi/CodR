Create `lib/theme/app_theme.dart` following the Material 3 pattern from RESEARCH.md Pattern 2:

```dart
import 'package:flutter/material.dart';
import 'package:google_fonts/google_fonts.dart';

const Color seedColor = Color(0xFF87CEEB); // Sky blue per D-01

ThemeData buildLightTheme() {
  final colorScheme = ColorScheme.fromSeed(
    seedColor: seedColor,
    brightness: Brightness.light, // D-02: light mode only
  );
  return ThemeData(
    useMaterial3: true,
    colorScheme: colorScheme,
    textTheme: GoogleFonts.interTextTheme(), // D-03: Inter font
  );
}
```

Update `lib/main.dart` to:
1. Import `app_theme.dart`
2. Wrap `QRScannerApp` with `MaterialApp` (still not `MaterialApp.router` — that comes in Plan 01-02)
3. Apply `theme: buildLightTheme()` to `MaterialApp`
4. Set `debugShowCheckedModeBanner: false`
5. Set `title: 'QR Scanner'`
6. Display a placeholder home screen with AppBar titled 'QR Scanner' to verify theme renders

Per D-04 (minimaliste/epure): keep the placeholder simple — just AppBar + centered text.
Per SPEC prohibition R2: DO NOT hardcode any colors — all colors must come from `Theme.of(context).colorScheme`. The seed color constant is allowed only in `app_theme.dart` where it feeds `ColorScheme.fromSeed`.
