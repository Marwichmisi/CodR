# Plan 05-02: Search/filter functionality and delete with confirmation

## Context

Plan 05-01 established the foundation: RecordBase abstraction, StorageService with getHistory() and FIFO, ScannerViewModel/GeneratorViewModel persistence integration, and a basic HistoryScreen with record list display. This plan adds the interactive features: real-time search, type filtering, and swipe-to-delete with confirmation dialog.

Key decisions from CONTEXT.md: D-16 (Dismissible + confirmDismiss pattern), UI-SPEC interaction contracts (search, filter, delete), UI-SPEC copywriting (French text for all UI elements).

## Tasks

### Task 1: HistoryViewModel search, filter, and delete capabilities

**Read First:**
- qr_scanner/lib/viewmodels/history_viewmodel.dart (created in Plan 05-01)
- qr_scanner/lib/services/storage_service.dart
- qr_scanner/lib/models/record_base.dart

**Files to Modify:**
- qr_scanner/lib/viewmodels/history_viewmodel.dart
- qr_scanner/test/viewmodels/history_viewmodel_test.dart

**Action:**

1. Add search state to `HistoryViewModel`:
   - New field: `String _searchQuery = ''`
   - New setter: `void setSearchQuery(String query)` that updates `_searchQuery` and calls `notifyListeners()`
   - New getter: `String get searchQuery => _searchQuery`

2. Add type filter state to `HistoryViewModel`:
   - Field already exists from Plan 05-01 (`_selectedType`)
   - Ensure setter `setFilter(RecordType?)` calls `notifyListeners()`

3. Add `filteredRecords` computed getter:
   - Start with `_allRecords`
   - If `_selectedType != null`, filter by `record.type == _selectedType!.name`
   - If `_searchQuery.isNotEmpty`, filter by `record.content.toLowerCase().contains(_searchQuery.toLowerCase())`
   - Return filtered list (already sorted by timestamp DESC from StorageService)

4. Update `deleteRecord()` method:
   - Determine table name: `record.type == 'scan' ? 'scan_records' : 'generation_records'`
   - Call `storageService.delete(table, record.id)`
   - Reload records via `loadRecords()` to refresh the list
   - Per D-06: no SnackBar notification on FIFO cleanup (silent)

5. Add tests to `test/viewmodels/history_viewmodel_test.dart`:
   - Test `setSearchQuery()` updates `searchQuery` and triggers `notifyListeners()`
   - Test `filteredRecords` with search query: only matching records returned
   - Test `filteredRecords` with type filter: only matching type returned
   - Test `filteredRecords` with both search and type filter combined
   - Test `filteredRecords` with empty search returns all records
   - Test `deleteRecord()` calls correct table based on record type
   - Test `deleteRecord()` refreshes list after deletion

**Acceptance Criteria:**
- [ ] `setSearchQuery()` updates search query and triggers listener notification
- [ ] `filteredRecords` returns only records matching search query (case-insensitive substring)
- [ ] `filteredRecords` returns only records matching selected type filter
- [ ] Search + filter combined: both conditions applied simultaneously
- [ ] Empty search query returns all records (no filtering)
- [ ] `deleteRecord()` determines correct table from record type
- [ ] `deleteRecord()` refreshes list after successful deletion
- [ ] Unit tests pass: `flutter test test/viewmodels/history_viewmodel_test.dart`

### Task 2: HistoryScreen search bar, filter chips, and swipe-to-delete

**Read First:**
- qr_scanner/lib/screens/history_screen.dart (created in Plan 05-01)
- qr_scanner/lib/viewmodels/history_viewmodel.dart (updated in Task 1 above)
- qr_scanner/lib/theme/app_theme.dart

**Files to Modify:**
- qr_scanner/lib/screens/history_screen.dart
- qr_scanner/test/screens/history_screen_test.dart

**Action:**

1. Add search bar to `HistoryScreen` (per UI-SPEC SearchBar contract):
   - Above the `ListView`, add `TextField` with:
     - `InputDecoration`: hint "Rechercher dans l'historique...", prefix icon `Icons.search`, clear button `Icons.clear` (visible when text non-empty)
     - Background color `#F4FAFC` via `fillColor`
     - `OutlineInputBorder` with `borderRadius: 12`
     - Padding `EdgeInsets.symmetric(horizontal: 16, vertical: 12)`
   - `onChanged` callback: call `widget.viewModel.setSearchQuery(value)`
   - Wrap in `Padding` with `EdgeInsets.symmetric(horizontal: 16, vertical: 8)`

2. Add type filter chips (per UI-SPEC TypeFilter contract):
   - `Row` of 3 `FilterChip` widgets: "Tout", "Scans", "Générations"
   - Default selected: "Tout" (no filter)
   - Selected chip: accent background (primary color at 20% opacity), accent border
   - Unselected chip: `#F4FAFC` background, grey border
   - Spacing: `SizedBox(width: 8)` between chips
   - `onSelected` callback: call `widget.viewModel.setFilter(RecordType?)` with appropriate value
   - Single selection: track selected chip index in local state
   - Wrap in `Padding` with `EdgeInsets.symmetric(horizontal: 16, vertical: 4)`

3. Add swipe-to-delete with `Dismissible` (per D-16, UI-SPEC Swipe-to-Delete):
   - Wrap each `Card` in `Dismissible` widget
   - `key: ValueKey(record.id)`
   - `direction: DismissDirection.endToStart` (swipe left only)
   - `background`: `Container` with `#D32F2F` (destructive red), `Alignment.centerRight`, `Icon(Icons.delete, color: Colors.white)`
   - `confirmDismiss`: Show `AlertDialog` (per UI-SPEC AlertDialog contract):
     - Icon: `Icons.delete_forever` tinted `#D32F2F`, size 48, above title
     - Title: "Supprimer l'entrée"
     - Content: "Voulez-vous vraiment supprimer cette entrée de l'historique ? Cette action est irréversible."
     - Actions: "Annuler" (text button, no action) + "Supprimer" (text button, `#D32F2F` color)
     - Barrier dismissible: `true`
     - Return `true` on confirm, `false` on cancel
   - `onDismissed`: call `widget.viewModel.deleteRecord(record)`

4. Add "no results" empty state (per UI-SPEC):
   - When `filteredRecords` is empty but `_allRecords` is not empty (search/filter active):
     - Show `Icons.search_off` size 64, color `onSurfaceVariant`
     - Heading: "Aucun résultat"
     - Body: "Aucune entrée ne correspond à votre recherche"

5. Update widget tests in `test/screens/history_screen_test.dart`:
   - Test search bar appears and accepts text input
   - Test typing in search bar filters the list (enter "example", only matching records shown)
   - Test clear button in search bar clears search and shows all records
   - Test filter chips appear and are selectable
   - Test tapping "Scans" filter shows only scan records
   - Test tapping "Générations" filter shows only generation records
   - Test tapping "Tout" filter shows all records
   - Test search + filter combined: both apply simultaneously
   - Test swipe-left on record shows red background
   - Test confirmDismiss shows AlertDialog with correct text
   - Test confirming deletion removes record from list
   - Test canceling deletion keeps record in list
   - Test "no results" state shows when search has no matches
   - Follow `flutter-add-widget-test` skill patterns

**Acceptance Criteria:**
- [ ] Search bar displays with hint "Rechercher dans l'historique..." and search/clear icons
- [ ] Typing in search bar filters records in real-time (case-insensitive)
- [ ] Clear button clears search and shows all records
- [ ] Three filter chips: "Tout", "Scans", "Générations" with correct selection states
- [ ] Tapping a filter chip filters records by type
- [ ] Search + filter combine: both conditions applied
- [ ] Swipe-left on record reveals red background with delete icon
- [ ] Releasing swipe shows confirmation AlertDialog with correct French text
- [ ] Confirming deletion removes record from list and database
